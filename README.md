# Risk File Checker — ระบบตรวจสอบเอกสาร PDF

ระบบตรวจสอบเอกสาร PDF ว่ามีเนื้อหาครบถ้วนตามข้อกำหนดที่ผู้ใช้กำหนดเอง
โดยใช้ Local LLM (Gemma 4 via Ollama) เพื่อรักษาความลับของข้อมูล — **ข้อมูลไม่ออกนอกเครื่อง**

---

## สิ่งที่ต้องติดตั้งก่อนใช้งาน

| ซอฟต์แวร์ | ลิงก์ดาวน์โหลด | หมายเหตุ |
|---|---|---|
| Python 3.10 ขึ้นไป | https://www.python.org/downloads/ | ตอนติดตั้งให้ **ติ๊กถูก "Add Python to PATH"** |
| Ollama | https://ollama.com | ติดตั้งแล้วเปิดทิ้งไว้ |

---

## วิธีใช้งาน

### 1. ดาวน์โหลดโปรเจกต์

คลิกปุ่ม **Code → Download ZIP** แล้วแตกไฟล์ หรือใช้ git:

```bash
git clone https://github.com/<your-username>/risk-file-checker.git
```

### 2. เปิดโปรแกรม

ดับเบิลคลิกไฟล์ **`start.bat`**

สคริปต์จะทำทุกอย่างให้อัตโนมัติ:
- สร้าง virtual environment
- ติดตั้ง library ที่จำเป็น
- ดาวน์โหลดโมเดล Gemma 4 (เฉพาะครั้งแรก อาจใช้เวลาสักครู่)
- เปิดเซิร์ฟเวอร์และเปิดเบราว์เซอร์ไปที่ http://localhost:8000

### 3. ใช้งานผ่านเบราว์เซอร์

1. กรอก **หัวข้อใหญ่** และ **หัวข้อย่อย** ที่ต้องการตรวจสอบ
2. อัปโหลดไฟล์ PDF
3. กด **ตรวจสอบ** — ระบบจะวิเคราะห์และแสดงผลว่าแต่ละหัวข้อผ่านหรือไม่ผ่าน

---

## การปิดโปรแกรม

กด `Ctrl + C` ในหน้าต่าง command prompt ที่รัน start.bat

---

## แก้ไขปัญหาเบื้องต้น

| ปัญหา | วิธีแก้ |
|---|---|
| `Python not found` | ติดตั้ง Python แล้วติ๊ก "Add Python to PATH" |
| `Ollama not found` | ติดตั้ง Ollama จาก https://ollama.com |
| เชื่อมต่อ Ollama ไม่ได้ | เปิดโปรแกรม Ollama ให้รันอยู่ก่อนกด start.bat |
| โมเดลช้ามาก | Gemma 4 ต้องใช้ RAM อย่างน้อย 8 GB — ถ้ามี GPU จะเร็วขึ้นมาก |
